QA Test Analysis: KAN-13 - Team Popularity Dashboard

URL: https://futbolbuggy.geekqa.net/

Criticality: HIGH

Goal: Implement and verify a real-time dashboard displaying team popularity rankings based on user votes.

1. Requirements Analysis

The core requirement is a "Standings" section on the main page that lists teams in descending order of votes. Key elements include team name, crest, and vote count. A critical technical requirement is that the counter must update in real-time after a vote is cast without a manual page refresh.

2. Risk Assessment

  • Real-time Update Failure: The most complex requirement; failure would mean users need to refresh to see their vote count.
  • Sorting Logic: Incorrect sorting (e.g., ascending instead of descending) would lead to a wrong leaderboard.
  • Responsive Layout: Dashboard components might break or become unreadable on mobile screens (375px).
  • Performance: Handling large vote counts ("large numbers") could impact rendering speed or data accuracy.

3. Decision Matrix & Skill Selection

Skill Status Rationale
generate-test-cases SELECTED Required to cover multiple ACs (Visibility, Visualization, Order, Real-time) and responsive scenarios.
execute-tests SELECTED Essential to verify the functional flow, especially the real-time update mechanism after voting.
run-accessibility-tests SELECTED Mentions "FutbolBuggy user or visitor" (inclusive) and "mobile devices". WCAG compliance is standard for public dashboards.
run-lighthouse-tests SELECTED Explicit concern in notes about "large numbers" and "high participation" suggests a need for performance auditing.
run-visual-tests SKIPPED No Figma files or design mockups were provided in the Jira ticket or asset folders.

4. Proposed Test Scenarios

  1. Visibility Check: Verify the "Dashboard" or "Standings" section is visible on load.
  2. Data Integrity: Ensure each entry has Name, Crest, and Vote count.
  3. Sorting: Verify teams are sorted by votes DESC.
  4. Functional Vote: Cast a vote for a team and verify the UI counter increments immediately without reload.
  5. Responsive UI: Verify the ranking layout at 375px, 768px, and 1440px.

5. Early Developer Recommendations

  • Implement unit tests for the sorting algorithm (ensure it handles equal votes correctly).
  • Verify that the websocket/polling event for vote updates only refreshes the specific DOM element to save resources.
  • Check that team crest images have proper `alt` text for screen readers.
