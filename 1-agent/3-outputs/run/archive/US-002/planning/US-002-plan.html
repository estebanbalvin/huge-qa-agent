Test Analysis: US-002 Inicio de sesión

1. Story Summary

Goal: As a registered user, I want to log in to access recipes and favorites.

URL: https://tusrecetas.lovable.app/

Acceptance Criteria

  • Valid credentials -> Session created (token) and redirect to feed.
  • Invalid credentials -> Generic error message (no user enumeration).
  • Session persistence until manual logout or expiration.

2. Risk Assessment

Criticality: HIGH - Access control is fundamental.

  • Lockout: Valid users cannot login.
  • Security: User enumeration via error messages.
  • Session: Token leakage or failure to persist.

3. Skill Selection Matrix

Skill Selected Rationale
Generate Test Cases ✅ YES Validation of positive and negative login flows.
Execute Tests ✅ YES Verify login functionality and redirection.
Run Accessibility ✅ YES Login inputs and errors must be accessible.
Run Lighthouse ❌ NO No performance requirements.
Run Visual Tests ❌ NO No design assets provided.

4. Proposed Test Scenarios

  • Happy Path: Login with valid confirmed account. Verify redirect to feed.
  • Negative: Login with wrong password. Verify generic error.
  • Negative: Login with non-existent email. Verify generic error.
  • Persistence: Reload page after login. Verify session remains active.

5. Developer Recommendations

  • Ensure error messages do not reveal if an email exists ("Invalid email or password").
  • Check HttpOnly flags on cookies/tokens if applicable.
